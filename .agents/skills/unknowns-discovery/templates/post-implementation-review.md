## Understanding Check

Answer these before merge or release:

1. What user-visible behavior changed?
2. Which old code paths now enter the new logic?
3. What are the three most likely failure modes?
4. Which conclusions are evidence-backed, and which remain assumptions?
5. How is the feature safely rolled back?
6. Where should a maintainer look first if this fails six months from now?
7. What data or external contracts could make future changes difficult?
