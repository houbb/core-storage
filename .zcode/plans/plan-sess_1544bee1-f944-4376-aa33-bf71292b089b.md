# P1 Metadata Runtime — 实现计划

## 概述

将 core-storage 从文件上传服务升级为**资源元数据中心**。新增 3 张数据表、元数据搜索 API、引用管理 API、前端元数据管理页面。P0 功能保持向后兼容（双写）。

---

## 一、数据库迁移（V2__create_metadata_tables.sql）

### 1.1 storage_metadata（核心元数据表）
新增字段 vs storage_file：`owner_type/owner_id/system_name/module_name/tags/remark/storage_driver/storage_key/resource_name/file_size`  
状态机：`UPLOADING` → `ACTIVE` → `REFERENCED` → `UNREFERENCED` → `SOFT_DELETED`

### 1.2 storage_reference（引用表）
字段：`metadata_uuid`, `system_name`, `module_name`, `business_type`, `business_id`  
索引：`metadata_uuid`, `(business_type, business_id)`

### 1.3 storage_metadata_index（轻量索引表）
字段：`resource_uuid`, `owner_type`, `owner_id`, `resource_type`, `module_name`, `tag`, `status`  
用途：为 P2/P3/P8 提供快速索引，避免扫描主表

所有表包含：`id, create_time, update_time, create_user, update_user`，禁止外键。

## 二、后端新建文件（共 16 个）

### Domain 层
- `StorageMetadata.java` — 领域对象（~22字段）
- `StorageReference.java` — 引用领域对象

### Entity 层
- `StorageMetadataEntity.java`
- `StorageReferenceEntity.java`
- `StorageMetadataIndexEntity.java`

### Converter 层
- `StorageMetadataConverter.java` — entity↔domain
- `StorageReferenceConverter.java` — entity↔domain

### Repository 层
- `StorageMetadataRepository.java` — save/findByUuid/search(多条件+分页+排序)/softDelete/updateStatus
- `StorageReferenceRepository.java` — save/findByMetadataUuid/deleteByUuid/countByMetadataUuid
- `StorageMetadataIndexRepository.java` — save/updateByResourceUuid

### Service 层
- `StorageMetadataService.java` — 核心方法：
  - `search(keyword,mimeType,status,hash,ownerType,ownerId,system,module,tag,startTime,endTime,sort,order,page,size)` → 分页结果
  - `getByUuid(uuid)` → metadata 详情
  - `createReference(metadataUuid,system,module,businessType,businessId)` → 创建引用 + 切状态
  - `deleteReference(referenceId)` → 删除引用 + 引用数为0时切 UNREFERENCED
  - `getReferences(metadataUuid)` → 引用列表

### Controller 层
- `StorageMetadataController.java`
  - `GET /api/v1/storage/metadata/{uuid}` — 元数据详情
  - `GET /api/v1/storage/metadata/search` — 多条件搜索（~12 查询参数）
  - `GET /api/v1/storage/metadata/{uuid}/references` — 引用列表
  - `POST /api/v1/storage/reference` — 创建引用
  - `DELETE /api/v1/storage/reference/{id}` — 删除引用

### Response DTO
- `StorageMetadataResponse.java` — 完整元数据字段
- `StorageReferenceResponse.java` — 引用信息
- `SearchResultResponse.java` — 分页包装

## 三、后端修改文件（共 3 个）

### StorageService.java
- `upload()` 在 `storage_file` 写入后，**同步双写** `storage_metadata`（status=UPLOADING→ACTIVE）
- 支持可选引用参数：system/module/businessType/businessId，传入则自动创建引用
- `delete()` 同步软删除 `storage_metadata`
- 同步写 `storage_metadata_index`

### StorageController.java
- `upload()` 新增可选 `@RequestParam`

### GlobalExceptionHandler.java
- 新增 `MetadataNotFoundException`、`ReferenceNotFoundException` 处理

## 四、前端新建文件（共 3 个）

### MetadataPage.vue
搜索栏：keyword + mimeType下拉 + status下拉 + 高级过滤折叠面板（ownerType/ownerId/system/module/tag/时间范围）  
结果表格：UUID | 文件名 | Mime | 大小 | Hash截断 | 状态badge | 引用数 | 时间  
分页：page+size  
点击行 → 打开右侧详情抽屉

### MetadataDetailDrawer.vue
左栏：文件信息（originalName, mimeType, fileSize, extension）  
右栏：元数据（UUID可复制, Hash可复制, StorageDriver, StorageKey, 状态, owner, system, module, tags, remark）  
底部：引用列表（system/module/businessType/businessId + 时间）

### stores/metadata.ts
- search(params) / fetchDetail(uuid) / fetchReferences(uuid) / createReference / deleteReference

## 八、关键设计决策（已确认）

| 决策 | 选择 |
|---|---|
| storage_file 处理 | 保留不动，上传双写两张表，P0 API 读旧表，P1 API 读新表 |
| metadata_index | P1 创建，上传时同步写入 |
| 上传引用 | 可选参数（system/module/businessType/businessId），传入即自动创建 |
| storage_key | 新增字段 + 保留 storage_name/relative_path |
| 状态机 | 5态完整实现，引用操作自动驱动状态迁移 |
| 前端 | 独立 MetadataPage + tab 切换 |

## 九、验证标准

1. `mvn test` 全部通过（新+旧 测试总计 ~25+ 用例）
2. 上传文件 → `storage_file` 和 `storage_metadata` 均有记录
3. 搜索 API 支持 keyword/mime/status/hash/owner/system/module/tag/time/sort/page
4. 创建引用 → metadata 状态切 REFERENCED；删除全部引用 → UNREFERENCED
5. 前端元数据页可搜索、浏览、查看详情、查看引用
6. 前端上传页功能不变（tab 共存）
